 33       
 34      To avoid confusion about the term "graph", we will use the following 
 35      naming convention: 
 36       
 37          * dgraph: a dot graph, i.e. an DotGraph instance (including any of its 
 38          subclasses) 
 39           
 40          * stgraph: a target/source graph, i.e. a DaesoGraph instance (including 
 41          subclasses like AlpinoGraph). 
 42           
 43          * agraph: an alignment graph, i.e. a GraphPair instance 
 44           
 45          * xgraph: a stgraph or agraph, i.e. any graph derived from networkx's 
 46          DiGraph class. 
 47           
 48      Analogously, we will use dnode, stnode, anode and xnode, as well as dedge, 
 49      stedge, aedge, xedge. 
 50       
 51      DotGraph provides update methods, like update_nodes or update_edges, which 
 52      can be called to create/update specific parts of the dot graph. A full 
 53      update is enforced by calling the update_structure method. Update methods 
 54      usually take one or more nx_graph arguments, representing a source/target 
 55      graph or a node alignment.
